# ⚖️ AI Legal Assistant - RAG with Google Gemini

[![GitHub Stars](https://img.shields.io/github/stars/anilbharath9/AI_Legal_Assisttant?style=social)](https://github.com/anilbharath9/AI_Legal_Assisttant)

An AI-powered legal document assistant that leverages Retrieval-Augmented Generation (RAG) with **Google Gemini 2.5 Flash** and **FAISS** for advanced legal reasoning and clause retrieval.

This assistant is designed to help users understand complex legal documents, retrieve relevant contract terms, and provide structured legal insights. By integrating Gemini 2.5 Flash, a sophisticated multimodal reasoning model, with the RAG framework, AI Legal Assistant ensures that responses are grounded in factual legal texts, reducing hallucinations and enhancing reliability. The assistant can process large legal documents, break them down into meaningful sections, and retrieve the most pertinent information to answer user queries accurately.

---

## 🚀 Features

*   📂 **Multi-Document Upload:** Process and analyze multiple legal PDF documents simultaneously.
*   🔍 **FAISS Vector Database:** Retrieve precise clauses and segments using local high-performance similarity search.
*   🤖 **Google Gemini Integration:** Generates highly context-grounded legal answers via dynamic API keys.
*   📜 **Contract Summarization & Obligation Analysis:** Extract terminations, governing laws, and liabilities.
*   📄 **Local Citation Expander:** Visually inspect matching document chunks directly in the Streamlit UI.

---

## 🛠️ Technologies Used

*   **Streamlit:** For building a fast, interactive, and responsive web user interface.
*   **LangChain:** Orchestrates document processing pipelines, prompt structures, and LLM chains.
*   **FAISS (Facebook AI Similarity Search):** A library for efficient similarity search and clustering of dense vectors locally.
*   **Google Gemini 2.5 Flash:** A high-speed, cost-effective reasoning model utilized for generating structured legal outputs.
*   **Sentence-Transformers (`all-MiniLM-L6-v2`):** Processes documents locally into 384-dimensional dense semantic vectors.
*   **PyPDF:** Extracts raw text blocks page-by-page from uploaded documents.

---

## ⚙️ How It Works

1.  **Ingestion & Load:** When legal PDFs are uploaded via the Streamlit interface, they are parsed using LangChain's `PyPDFLoader` to extract raw textual layouts.
2.  **Semantic Chunking:** Text is broken into smaller logical segments (chunk size 300, overlap 80) via `RecursiveCharacterTextSplitter` to fit LLM context limits and retain sentence coherence.
3.  **Local Embedding Generation:** Chunks are vectorized using a local Sentence Transformer model (`all-MiniLM-L6-v2`).
4.  **Vector Indexing:** High-dimensional embeddings are cached and stored inside a local **FAISS** index directory (`vectorstore/faiss_index`).
5.  **Context-Aware Retrieval:** When a query is submitted, FAISS conducts a local similarity search, returning the top **K=3** most relevant legal document chunks.
6.  **Prompt Construction & LLM Generation:** The query and matching text chunks are combined into a strict legal instruction prompt template. This payload is processed by the **Google Gemini 2.5 Flash** API, delivering a factual response containing specific references.

## 🚀 Installation & Setup

### Prerequisites

*   Python 3.10 or higher
*   Google Gemini API Key (Obtain from [Google AI Studio](https://aistudio.google.com/))

### 1. Setup Virtual Workspace

```bash
# Clone repository
git clone https://github.com/anilbharath9/AI_Legal_Assisttant.git
cd AI_Legal_Assisttant

# Create environment
python -m venv venv

# Activate workspace
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 2. Install Requirements

```bash
pip install -r requirements.txt
```

### 3. Configure Variables

Create a `.env` file in your project root:

```env
GOOGLE_API_KEY=your_actual_api_key_here
```

### 4. Launch Application

```bash
# Start Streamlit application
streamlit run app.py
```

Or pre-process document directories beforehand:

```bash
python create_embeddings.py
```
